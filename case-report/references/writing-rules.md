# Academic English Writing Rules for Case Reports

症例報告原稿に適用する英語執筆ルール。ドラフト作成時およびセルフチェック時に参照する。

---

## 13 Core Rules

### 1. Terminology consistency（用語統一）
- 1つの概念に対して1つの英語表現を文書全体で統一する
- 略語は初出時に定義し、以降一貫して使用する
- 大文字・小文字の使い分けを統一する

### 2. Claim strength calibration（主張の強度調整）
- 症例報告では因果関係の主張を避ける。"caused" ではなく "was associated with", "may have contributed to" を使う
- 確定的な表現（"proved", "definitely", "breakthrough"）を避ける
- 一般化を避ける。"This case suggests..." のように限定する

### 3. Specificity（具体性）
- 曖昧な量的表現（many, several, various）を具体的な数値に置き換える
- 検査値には単位を必ず付記する
- 時間経過は具体的に記述する（"数日後" ではなく "3 days later"）

### 4. Clear reference（明瞭な照応）
- 裸の "this/these/that" を避ける。必ず名詞を付ける: "this finding", "these results"

### 5. Conciseness（簡潔性）
- 冗長表現を短縮する:
  - "in order to" → "to"
  - "due to the fact that" → "because"
  - "a total of N patients" → "N patients"
  - "it is worth noting that" → 削除
- フィラーを削除: "It is important to note that", "Interestingly", "Notably"

### 6. Limit nominalization（名詞化の抑制）
- 動詞を名詞化した表現を避ける
  - BAD: "evaluation of improvement of detection"
  - GOOD: "evaluating how detection improved"
- 3語以上の名詞連鎖は再構成する

### 7. Subject-verb proximity（主語-動詞の近接）
- 主語と主動詞の間が10語を超えたら再構成する
- 修飾句を関係節や前置詞句に移動するか、文を分割する

### 8. Voice consistency（態の一貫性）
- Case Presentation: 受動態が一般的（"The patient was admitted..."）だが能動態も可。セクション内で統一する
- Discussion: 能動態を推奨。段落内で態を混在させない

### 9. Connector discipline（接続詞の機能的選択）
- 論理機能に基づいて選ぶ:
  - Addition: In addition, Moreover, Also
  - Contrast: However, In contrast, Although
  - Cause/effect: Therefore, Thus, Consequently
  - Example: For example, Specifically
  - Summary: In summary, Overall
- "However" はセクション内で2回まで
- 連続する段落を同じ接続詞で始めない

### 10. One idea per sentence（1文1アイデア）
- 所見とその解釈を1文に詰め込まない。分割する
- 2つの独立した所見がある場合も分割する

### 11. Sentence length（文長）
- 目標: 平均15-25語/文
- 上限: 約35語。超えたら分割する
- 従属節は1文に1つまで

### 12. Paragraph structure（段落構成）
1. Topic sentence（段落の主張）
2. Evidence（根拠・データ）
3. Interpretation（必要な場合のみ）
4. Closing/Transition（次段落への橋渡し）

### 13. Tense consistency（時制の一貫性）
- Introduction: 確立された知識は現在形、特定の先行研究は過去形
- Case Presentation: 過去形
- Discussion: 一般的主張は現在形、自症例の結果は過去形

## AI-Phrasing Avoidance List

以下の表現はAI生成テキストに典型的であり、使用を避ける:

| 避ける表現 | 代替表現 |
|---|---|
| utilize | use |
| demonstrate | show |
| elucidate | clarify, explain |
| facilitate | help, enable |
| comprehensive | thorough, full |
| robust | strong, reliable |
| leverage | use, apply |
| underscore | emphasize, highlight |
| pivotal | key, central |
| delve into | examine, explore |
| cutting-edge | advanced, latest |
| groundbreaking | novel, new |
| it is important to note | (削除) |
| it should be noted that | (削除またはリフレーズ) |
| in recent years | 具体的に: "Since 2020" 等 |
| a growing body of evidence | 具体的に引用する |

## Self-Check Checklist

ドラフト完成時に以下を確認する:
- [ ] "this/these/that" に必ず名詞が付いている
- [ ] 35語を超える文がない
- [ ] 1段落に複数の主張が混在していない
- [ ] 用語が文書全体で統一されている
- [ ] 因果関係の主張を避けている（症例報告として適切な表現）
- [ ] AI典型表現が残っていない
- [ ] 接続詞が多様で機能的に適切
- [ ] 主語-動詞間が10語以内
- [ ] 時制がセクション内で一貫している
- [ ] 自然で機械的でない文章になっている
